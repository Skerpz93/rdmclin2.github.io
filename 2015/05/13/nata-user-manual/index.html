NATA 用户手册

發表於 2015-05-13   |   分類於 test   |  

前言

本文为NATA用户手册,为使用该工具的用户提供帮助

环境配置

  • 安装JDK 1.7及以上，JDK下载网址

  • 安装Android SDK ,最新的即可，用SDK Manager下载platform-tools和tools，以及需要的platforms,参考下载地址:
    Win32，
    Win64

  • 安装ANT,下载地址，下载后解压即可。

  • 将提供的android-21文件夹粘贴到sdk的platforms文件夹下（因为我们的测试apk的版本是21）

设置系统环境变量：

将tools和platform－tools，以及build－tools的某一个版本文件夹加入到path路径中,将ant的bin目录加入到path路径中

在cmd中输入ant测试ant环境是否配置正确
输入adb查看是否有help提示
输入aapt查看是否有help提示

使用

NATA默认读取 $ANADROID_HOME$ 环境变量作为sdk目录，sdk目录中的platforms为插桩apk时所需要使用的平台库。
NATA默认不进行soot方法插桩，因为插桩对于大型apk不一定插桩成功。
注意：我们假设apk的目标平台在sdk的platforms文件夹中存在！如果不存在，请用SDK Manager下载，或者看下面的问题5。

在使用本工具之前，请确保设备连接手机,在cmd中输入

观察是否设备已连接上。

基本使用

带方法插桩的测试

默认我们认为目标apk所需要使用的安卓平台在sdk的platfroms中存在，如果不存在，请下在对应平台到platforms文件夹中

可配置参数

  • --apk 对象apk的路径，或者工程源代码（源代码目前对有库函数的工程支持不好，最好不要使用）
  • --output 输出路径,默认路径为< 当前目录/nata-output >
  • --sdk ANDROID SDK的路径，默认读取 $ANADROID_HOME$ 环境变量
  • -h or --help 显示帮助信息
  • -s 开启方法插桩，方法插桩有几率失败，默认不开启

查看结果

测试结果存放在nata-output/SpoonRunner/SpoonOutput中，文件结构如下：

  • ActivityCoverages activity覆盖率的统计报告
  • image 测试过程中的截图
  • junit-reports junit的测试报告，可查看测试运行情况，如测试时间等等
  • logs 测试过程中产生的log，包括运行到的方法的统计（如果开启了-s选项的话)
  • MethodCoverages 方法覆盖率的统计报告(如果开启－s选项的话)
  • model 测试工程的中间结果,可忽略
  • web 产生的树状遍历图。

常见错误处理

如果出现错误，请看下面的内容是否能够帮助您：
1.Unsupported major.minor version 51.0 error?

答：请检查你的jdk版本，至少需要jdk1.7,请下载1.7的最新版本。

2.Cannot run program “zipalign”: CreateProcess error=2, ?????????
INFO: signapk & zipalign returned:jar 已签名。警告: 未提供 -tsa 或 -tsacert, 此
jar 没有时间戳。如果没有时间戳, 则在签名者证书的到期日期 (2045-03-04) 或以后的任
何撤销日期之后, 用户可能无法验证此 jar。null

答：是否忘记把build-tools文件夹加入到PATH路径中了？

3.[echo] Resolving Build Target for Test…BUILD FAILEDD:\adt\sdk\tools\ant\build.xml:542: Unable to resolve project target ‘android-21’Total time: 0 seconds

答：是否忘记将android－21放到platforms文件夹中？

4.app apk install failed. Error[INSTALL_PARSE_FAILED_INCONSISTENT_CERTIFICATES]?

答：因为不同机器上产生的keystone不同，如果是新的pc,目标手机上先前安装过该应用，需要将其先卸载。注意测试apk［smartTest]也需要一并卸载，可以在设置－》应用中卸载。

5.Caused by: java.lang.RuntimeException: error: target android.jar (C:\Program Fil
es\Android\android-sdk\platforms\android-4\android.jar) does not exist.

答：如果开启了－s（即代码插桩选项），那么sdk的platforms文件夹中应当包含这个apk的目标平台的platforms，我们整理了安卓各平台（api3-api21）的platform文件，可以在github上选择缺失平台下载，然后粘贴到sdk的platforms文件夹下。

6.INFO: signapk & zipalign returned:jarsigner 错误: java.lang.RuntimeException: 密
钥库加载: C:\Users\Lenovo.android\debug.keystore (系统找不到指定的文件。)
ERROR: Problem occured while signing apk
INFO: Error occured while doInstrumentation with testApp

答：如果在机器上用安卓环境编译过apk，那么在用户的.android文件夹下会生成默认的debug.keystone,如果该目录下没有该文件，请创建一个新的anroid工程并进行编译。
